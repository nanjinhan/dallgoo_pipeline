# 영상 녹화 수집 파이프라인 — 학습 정리 노트

> 이 프로젝트를 만들면서 나온 **전체 흐름 + 용어 뜻 + 헷갈렸던 개념**을 한 곳에 정리한 복습용 노트.
> 나중에 다시 볼 때 이 파일 하나만 보면 "뭘 만들었고, 어떻게 돌아가고, 무슨 단어가 무슨 뜻인지" 파악할 수 있게 정리함.

---

## 0. 이 프로젝트가 하는 일 (한 줄)

**카메라로 녹화 → 세션 단위로 저장 → 무손실 압축 → VPN 통해 회사 NAS로 자동 전송 → 무결성 검증.**

교육장에 설치해서, 사람은 낮에 녹화만 하고 밤에 스케줄러가 알아서 NAS로 보내는 게 목표.

---

## 1. 전체 그림 (아키텍처)

```
[카메라] ──RTSP──▶ [go2rtc] ──┬──▶ 브라우저 라이브뷰 (표시 전용)
                              └──▶ 백엔드가 stream.mp4 받아 디스크에 녹화
                                        │
                                        ▼
                              recordings/세션폴더/ (cam_1.mp4 + packet.json + checksums)
                                        │  (녹화 중지 시 자동 압축)
                                        ▼
                                   세션폴더.zip
                                        │  (스케줄러가 정한 시각에)
                            VPN 연결 → SFTP 업로드 → SHA-256 검증 → sent/ 이동
                                        ▼
                              [회사 NAS] /training-recordings/pipetest1/
```

**구성요소 3개**
| 구성 | 역할 | 주소 |
|---|---|---|
| go2rtc | 카메라 영상을 받아 나눠줌(라이브뷰 + 녹화용) | localhost:1984 |
| 백엔드(FastAPI) | 녹화 제어·저장·압축·전송 | localhost:8000 |
| 웹페이지 | 라이브뷰 + 녹화 버튼 + 타이머 | 브라우저 |

---

## 2. 전체 데이터 흐름 (단계별)

### ① 녹화
- 웹에서 **녹화 시작** → 백엔드가 go2rtc의 `stream.mp4`를 받아 임시폴더 `_tmp_...`에 기록 (카메라당 1스레드)
- **녹화 중지** → 임시폴더를 정식 이름 `시작시각__종료시각`으로 확정

### ② 세션 저장 (한 세션 = 녹화 한 묶음)
폴더 안에 3개 생성:
- `cam_1.mp4` — 실제 영상 (카메라 원본 코덱 그대로)
- `packet.json` — 세션 메타(시작/종료 시각, 카메라 목록) + 마네킹 패킷 자리(빈 배열)
- `checksums.sha256` — 폴더 안 각 파일의 무결성 지문

### ③ 자동 압축 (중지 즉시)
- 중지하면 백그라운드로 폴더를 **무손실 ZIP**으로 압축 → `세션폴더.zip`
- zip의 SHA-256도 계산 (전송 후 대조용)
- 웹에 "⏳ 압축중… → ✅ 압축 완료" 표시

### ④ 전송 (스케줄러 or send.bat)
1. **VPN 연결 확인** (NAS `10.8.0.1:2222` 도달되나)
2. **SFTP 업로드** (개인키로 로그인)
3. **SHA-256 무결성 검증** — 올린 zip을 다시 읽어 해시 대조 (전송 중 안 깨졌나)
4. **성공 → zip을 `sent/`로 이동 + 원본 폴더 삭제** / **실패 → 폴더 남겨 다음에 재시도**

## 3. 저장·보관 규칙 (헷갈렸던 부분)

### 시점별로 로컬에 뭐가 남나
| 시점 | 로컬 상태 |
|---|---|
| 녹화 중지 직후 | **폴더 + zip** 둘 다 (아직 전송 전) |
| 전송 성공 후 | **zip만** (`sent/`에) — 원본 폴더 삭제 |
| 전송 실패 | 폴더 그대로 (다음 주기 재시도) |

**핵심**: "원본 폴더 삭제"는 **전송 성공한 뒤**에만 일어남. 전송 안 한 세션은 폴더가 남아있는 게 정상.

### 왜 zip만 남기나
- zip은 **무손실**이라 풀면 원본 100% 복구 가능
- 진짜 원본은 이미 NAS에 검증까지 마치고 올라가 있음
- 그래서 로컬엔 압축본만 백업으로 남기고 원본은 지워 디스크·중복 절약

## 4. 스케줄러가 5개 세션을 처리하는 실제 흐름 (예시)

```
[낮] 녹화 5번 → 각자 중지 시 자동 압축
     recordings/ 에 (폴더+zip) × 5

[새벽 3시] 작업 스케줄러 발동 → run_transfer.py
  ① 안 보낸 세션 수집 (sent/ 안은 제외)
  ② VPN 도달 확인 (안 되면 전부 스킵, 내일 재시도)
  ③ 하나씩:
       빈 세션(영상 없음)? → 스킵
       정상 → 압축(있으면 재사용) → 업로드 → SHA-256 검증 → sent/ 이동 + 폴더 삭제
       실패 → 폴더 남겨 재시도
```

**중요 성질**
- **idempotent**: 몇 번을 돌려도 이미 보낸 건 안 보냄, 실패한 것만 재시도
- **빈 세션 가드**: 영상 없는 세션은 압축·전송 안 함
- **하나 실패해도 전체 안 멈춤**

## 5. 용어 사전 (무슨 단어가 무슨 뜻)

### 영상 관련
| 단어 | 뜻 |
|---|---|
| **go2rtc** | 카메라 영상을 받아 여러 곳(라이브뷰+녹화)에 나눠주는 프로그램 |
| **RTSP** | CCTV·IP카메라가 영상을 내보내는 표준 프로토콜 (주소가 `rtsp://...`) |
| **remux(리먹스)** | 영상 코덱은 그대로 두고 **포장(컨테이너)만** 바꾸는 것. 화질 손실 없음 |
| **재인코딩(transcoding)** | 영상을 **다시 압축**하는 것. 화질 손실 발생 → 이 프로젝트는 **금지** |
| **코덱** | 영상 압축 방식 (예: H.264). C920 웹캠은 H.264를 자체 출력 |
| **fan-out** | 하나의 카메라 영상을 여러 곳으로 **동시에 나눠주는** 것 |
| **WebRTC** | 브라우저에서 저지연으로 실시간 영상 보는 기술 (라이브뷰에 사용) |
| **ffmpeg** | 영상 처리 만능 도구. go2rtc가 웹캠 잡을 때 사용 |
| **스트림(stream)** | 실시간으로 흘러오는 영상 데이터 |

### 저장·압축 관련
| 단어 | 뜻 |
|---|---|
| **세션(session)** | 녹화 시작~중지 **한 묶음**. 세션마다 폴더 하나 |
| **packet.json** | 세션 설명서(메타데이터). 언제·어떤 카메라로 찍혔는지 + 마네킹 데이터 자리 |
| **무손실 압축** | 풀면 원본과 100% 똑같은 압축 (ZIP/Deflate). 화질·데이터 손실 없음 |
| **SHA-256** | 파일의 "지문". 파일이 1비트라도 바뀌면 값이 완전히 달라짐 |
| **체크섬(checksum)** | 무결성 확인용 값. 여기선 SHA-256 해시 |
| **무결성(integrity)** | 파일이 전송 중 깨지거나 변조되지 않았음 |
| **idempotent(멱등)** | 여러 번 실행해도 결과가 같음. 중복 전송 없음 |

### 네트워크·보안 관련 ⭐ (제일 헷갈렸던 부분)
| 단어 | 뜻 |
|---|---|
| **VPN** | 인터넷 너머 사설망으로 들어가는 **"통로"**. 켜면 NAS가 `10.8.0.1`로 보임 |
| **SSH** | 원격 서버에 안전하게 로그인하는 방식 |
| **SFTP** | SSH 위에서 도는 **파일 전송** (Secure FTP) |
| **개인키(private key)** | 인증하는 **열쇠 파일**. 나만 보관, 절대 공유 금지 (`nas_pipetest1`) |
| **공개키(public key)** | 서버에 등록하는 **자물쇠**. 남한테 줘도 안전 (`.pub`) |
| **passphrase** | 개인키 파일 자체에 걸린 **비밀번호** (GitLab 로그인 비번과 무관!) |
| **DDNS** | 유동 공인IP를 고정 주소로 접속하게 해주는 것 (`dallgoo.synology.me`) |
| **핫스팟** | 휴대폰 데이터로 인터넷 연결 (외부망 테스트용) |
| **chroot** | 계정을 특정 폴더 안에 가둠. 그래서 NAS 경로가 `/volume1` 없이 `/training-recordings/...` |
| **NAS** | 네트워크 저장장치 (여기선 Synology DS925+) |

### 도구·자동화
| 단어 | 뜻 |
|---|---|
| **FastAPI** | 파이썬 백엔드 웹 프레임워크 |
| **Windows Task Scheduler** | 정한 시각에 프로그램 자동 실행하는 윈도우 기능 |
| **스레드(thread)** | 동시에 여러 작업 처리하는 단위. 카메라마다 1스레드로 녹화 |
| **paramiko** | 파이썬에서 SFTP 하는 라이브러리 |
| **.env** | 비밀·설정값을 코드와 분리해 담는 파일 (git 제외) |

## 6. 자주 헷갈렸던 개념 Q&A ⭐

### Q. VPN과 SSH는 무슨 관계?
**VPN = 통로(길), SSH = 그 통로 끝에서 문 열고 파일 넣기.**
- VPN을 켜면 NAS까지 **닿을 수 있게** 됨 (네트워크 진입)
- 그다음 SSH 개인키로 **로그인해서 파일 전송**
- 자물쇠 2개: VPN 뚫어도 SSH 키 없으면 못 들어감 (이중 보안)

### Q. 공개키는 남한테 알려줘도 돼?
**응, 돼.** 공개키는 원래 남 주라고 만든 것(=자물쇠). 개인키(열쇠)만 안 주면 됨.

### Q. 지금 필요한 게 무슨 키야?
**개인키.** 근데 이미 PC에 있음. 새로 구할 키는 없고, 필요하면 그 개인키를 **여는 비밀번호(passphrase)** 를 치는 것뿐.

### Q. passphrase가 GitLab 로그인 비번이야?
**아니.** 완전 별개. passphrase는 **키 파일에 걸린 잠금 비번**. GitLab 웹 로그인 비번과 무관.

### Q. 원본 폴더가 남아있는데? zip만 남는다며?
- 전송 **안 한** 세션 = 폴더 + zip 둘 다 있음
- 전송 **성공한** 세션 = zip만 (`sent/`) — 이때 원본 폴더 삭제
- 즉 "폴더 삭제"는 전송 성공 후에만 일어남

### Q. 전송은 내가 직접? 자동?
- **무엇을 보낼지** = 자동 (안 보낸 것 전부, 하나씩 안 고름)
- **언제 보낼지** = 수동(`send.bat`) 또는 자동(스케줄러 정한 시각)

### Q. SHA 검증은 뭘 검증해?
**zip 압축본 전체.** 로컬 zip 해시 = NAS에 올라간 zip 해시 인지 대조 → 전송 중 안 깨졌나 확인.

### Q. 다른 노트북에서 써도 흐름 같아?
**똑같음.** VPN 켜면 어디서든 NAS는 `10.8.0.1`로 보임. 바뀌는 건 `.ovpn`의 접속주소(사내=내부IP / 외부=DDNS)와 계정뿐. 코드는 그대로.

## 7. 실전 사용법 (명령어)

### 서버 켜고 끄기
```
start.bat  더블클릭  → go2rtc + 백엔드 켜고 브라우저 열림
stop.bat   더블클릭  → 둘 다 종료
```
- 캠만 안 보이면(서버는 켜짐) → 브라우저 새로고침

### 녹화
- 브라우저 `http://127.0.0.1:8000` → 녹화 시작 / 중지 (오른쪽 상단에 녹화 시간 실시간 표시)

### 전송 (VPN 켠 상태에서)
```
send.bat  더블클릭
또는
python -m scheduler.run_transfer
```

### 자동 전송
- Windows 작업 스케줄러 `recording-pipeline-transfer` = 매일 새벽 3시 자동
- VPN은 OpenVPN Connect "자동 연결"로 상시 켜두기 (A방식)

### 코드 수정 후 GitLab 올리기
```powershell
cd c:\Users\User\Desktop\pipeline
git add -A
git commit -m "무엇을 바꿨는지"
git push
```

## 8. 파일별 역할

```
backend/
  config.py    — .env 읽어서 설정값 제공
  recorder.py  — go2rtc 영상을 파일로 기록 (카메라당 1스레드)
  main.py      — 웹 API (/start /stop /status), 중지 시 자동 압축
  session.py   — 세션 폴더 확정 + packet.json + checksums 생성
  archive.py   — 무손실 ZIP 압축 + SHA-256 (이미 있으면 재사용)
  vpn.py       — VPN 연결 확인 / 필요시 자동 연결·해제
  transfer.py  — SFTP 업로드 + 무결성 검증 + sent 이동 + 빈세션 가드
  logutil.py   — 로그 기록
scheduler/
  run_transfer.py  — 무인 배치 (수집→VPN→전송)
  run_transfer.bat — 작업 스케줄러가 실행하는 래퍼
go2rtc.yaml   — 카메라 스트림 정의
web/index.html — 라이브뷰 + 녹화 버튼 + 타이머
.env          — 실제 설정값 (git 제외, 비밀)
.env.example  — 설정 양식 (값 없이 구조만, git 포함)
start/stop/send.bat — 원클릭 실행 스크립트
```

## 9. 겪었던 문제 & 해결 (트러블슈팅)

| 증상 | 원인 | 해결 |
|---|---|---|
| 웹에 캠 화면 안 뜸 | go2rtc 미실행 | go2rtc.exe + ffmpeg.exe 설치·실행 |
| `cam_1 열 수 없음 0x80004005` | GPU 하드웨어 인코딩 오류 | go2rtc.yaml에서 `#hardware` 제거(소프트웨어 인코딩) |
| 녹화했는데 cam_1.mp4 없음 | go2rtc 켜기 전에 녹화함(빈 세션) | go2rtc 먼저 켜고 녹화 |
| 녹화 시간 타이머 안 보임 | 브라우저가 옛 페이지 캐시 | 백엔드에 캐시 금지 헤더 추가 |
| 빈 zip이 NAS에 올라감 | 옛날엔 크기만 검증(빈 파일끼리 통과) | SHA-256 검증 + 빈 세션 가드 추가 |
| SFTP 경로 `/volume1/...` 없음 | 계정이 chroot됨 | `/training-recordings/pipetest1` (앞에 /volume1 빼기) |
| git push `Permission denied` | 개인키에 passphrase 걸림 | 비번 없는 새 키 생성·등록 |
| git push `default branch...pre-receive declined` | GitLab 프로젝트 브랜치 보호/권한 | 보호 해제 or 본인 계정 밑에 생성 |
